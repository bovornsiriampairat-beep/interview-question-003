## 📋 สรุปแนวทางการแก้ไขปัญหาและการพัฒนา (Development Summary)
ในการทดสอบนี้เป็นการเชื่อมต่อระบบแบบแยกหน้าบ้านและหลังบ้าน (Full-Stack Separated Application) โดยได้ทำการปรับปรุงและแก้ไขส่วนต่างๆ เพื่อให้ระบบสามารถทำงานร่วมกันได้อย่างสมบูรณ์ตามโจทย์กำหนด ดังนี้:

### 1. การแก้ไขโครงสร้างหน้าบ้าน (Angular Frontend)
* **การปรับใช้สถาปัตยกรรมแบบ Signals**: ปรับเปลี่ยนโครงสร้างคลาสหลักจาก `app.component.ts` เดิม ไปรันผ่านโครงสร้าง `App` ตัวใหม่ (`app.ts`, `app.html`) ที่สร้างมากับ Template ระบบ และเลือกใช้ **Angular Signals (`signal`)** เพื่อจัดการ State ข้อมูลตาราง ซึ่งช่วยให้หน้าจอทำการเรนเดอร์ข้อมูลใหม่ได้ทันที (Reactive Change Detection)
* **การตั้งค่า Dynamic Environment**: สร้างไฟล์โครงสร้างการตั้งค่าระบบไว้ที่ `public/assets/config.json` เพื่อแยกตัวแปร URL พอร์ตของ API ออกจากโค้ดระบบ สะดวกต่อการเปลี่ยนสภาพแวดล้อมในการทดสอบ
* **การปรับปรุงดีไซน์ (UI/UX)**: 
  * ออกแบบหน้าเพจให้มีลักษณะเป็นกลาง (Generic UI) ไม่ระบุชื่อบริษัทหรือองค์กรใดๆ เพื่อความเป็นสากล
  * เลือกใช้ชุดฟอนต์ **Google Sans** ร่วมกับ Open Sans เพื่อความสวยงาม อ่านง่าย และเป็นระเบียบเรียบร้อย
  * ปรับสไตล์สีปุ่มในหน้าต่างยืนยันย่อย (Modal Windows) ให้ถูกต้องตรงตามเงื่อนไขของโจทย์ โดยปุ่มตกลงทำรายการหลัก (ทั้งอนุมัติและไม่อนุมัติ) จะใช้ **สีเขียว** และปุ่มยกเลิกจะถูกจัดวางไว้ขวาสุดด้วย **สีแดง**

### 2. การเชื่อมต่อระบบหลังบ้าน (.NET Backend API)
* ทำการเปิดทดสอบเซิร์ฟเวอร์ .NET Web API ผ่านพอร์ตสตรีมข้อมูลจริง `http://localhost:5184`
* แก้ไขปัญหาการบล็อกรีเควสข้ามพอร์ตระหว่างพอร์ตหน้าบ้าน (`:4200`) และหลังบ้าน (`:5184`) โดยการเปิดนโยบายสิทธิ์ความปลอดภัย **CORS (Cross-Origin Resource Sharing)** ในไฟล์ `Program.cs` ฝั่งหลังบ้าน ทำให้การรับส่ง Data Array ทั้ง 10 รายการเสร็จสิ้นสมบูรณ์

## 🛠️ ขั้นตอนการรันเพื่อทดสอบระบบ (How to Run & Test)

### 1. การรันระบบหลังบ้าน (Backend Web API)
1. เปิด Terminal เข้าไปที่โฟลเดอร์โปรเจกต์หลังบ้าน:
   ```bash
   cd Com.Example.Approval.Api
2. เปิด Terminal เข้าไปที่โฟลเดอร์โปรเจกต์หลังบ้าน:
   ```bash
   dotnet run
2. การตั้งค่าและรันระบบหน้าบ้าน (Angular UI)
เปิดไฟล์ example-approval-ui/public/assets/config.json เพื่อตรวจสอบหรือปรับแก้พอร์ต API ให้ชี้ไปที่หลังบ้าน:

JSON
{
  "apiUrl": "http://localhost:5184"
}
เปิด Terminal อีกหน้าต่างย่อยเข้าไปที่โฟลเดอร์หน้าบ้าน ยิงคำสั่งเพื่อติดตั้งคลังคำสั่งพื้นฐาน:

Bash
npm install
สั่งรันหน้าเว็บแอปพลิเคชันด้วยคำสั่งมาตรฐาน:

Bash
npx ng serve
เปิด Browser เข้าใช้งานที่ลิงก์: http://localhost:4200

💡 สรุปสถานะการทำงานของหน้าจอ (Flow State Definition)
หน้าจอหลัก (IT 03-1): แสดงรายการตารางเอกสารทั้งหมด 10 รายการที่ดึงมาจากฐานข้อมูลหลังบ้าน โดยปุ่ม "อนุมัติ" และ "ไม่อนุมัติ" ด้านบนจะเปิดให้คลิกได้ก็ต่อเมื่อมีการเลือกติ๊ก Checkbox บนรายการที่เป็นสถานะ "รออนุมัติ" เท่านั้น ส่วนรายการที่เป็น "อนุมัติ" หรือ "ไม่อนุมัติ" แล้ว จะแสดงเป็นกล่องทึบสีเทาเพื่อล็อกไม่ให้เลือกซ้ำ

หน้าจอยืนยันการอนุมัติ (IT 03-2): แสดงผลเมื่อกดปุ่มอนุมัติ ระบบจะเปิด Modal Popup ให้ระบุเหตุผลประกอบ พร้อมปุ่มยืนยันงานสีเขียว

หน้าจอยืนยันการไม่อนุมัติ (IT 03-3): แสดงผลเมื่อกดปุ่มไม่อนุมัติ ระบบจะเปิด Modal Popup ให้กรอกเหตุผลประกอบ พร้อมปุ่มยืนยันงานสีเขียว และปุ่มยกเลิกสีแดงขวาสุด
